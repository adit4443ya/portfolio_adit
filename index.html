The Work That Defined the Year

Google Summer of Code 2025 · Fortran-Lang · May–Sept

OpenMP 6.0 Infrastructure & GPU Offloading in LFortran

C++ · Fortran · OpenMP · CUDA · LLVM / ASR

LFortran is a from-scratch Fortran compiler targeting LLVM and C backends. My GSoC project built OpenMP deep enough to eventually enable GPU execution of Fortran — something no open-source Fortran compiler handles cleanly today.

The significant architectural decision: OpenMP handling was entangled with the DoConcurrentLoop path. I proposed and implemented a dedicated OMPRegion ASR node — giving OpenMP first-class representation in the IR tree. This unlocked structurally clean handling of nested and hierarchical parallelism that was architecturally impossible before.

With the IR foundation in place: 13+ constructs, 8+ clauses spanning thread, team, and task models — then extended the C-backend to emit compilable host-device code for OpenMP Target Offloading on NVIDIA GPUs. A custom GPU emulator (<250 lines) handles CI without physical hardware.

LFortran Compiler — Core Contributor

C++ · Fortran · MPI · ISO_C_BINDING

Contributed to compiling POT3D — Predictive Science's MPI + OpenMP solar magnetic field solver used in real space weather research — with LFortran. The 9th production-grade third-party code LFortran had ever compiled, a public milestone for the project.

Built a pure Fortran MPI wrapper library using ISO_C_BINDING with 30+ subroutine implementations — eliminating C-wrapper overhead entirely. Now lives in the fortran_mpi repo under the lfortran org. Separately: 50+ compiler issues resolved across OpenMP, OOP, structs, and strings.
